# Design: 电视远程管控 App

Generated by /office-hours on 2026-05-16
Repo: 电视控制APP
Status: DRAFT
Mode: Builder

## Problem Statement

家长离家 40 公里时，无法控制家里孩子看电视的时间和内容。电视开机后自动联网，孩子可以无限制观看。即使远程关闭电视，孩子重启后又能继续看。

## What Makes This Cool

- 家长在手机上点一下「关闭」，电视立刻黑屏，重启也绕不过
- 不依赖额外硬件，电视自己管自己
- 鸿蒙手机 + 华为智慧屏原生生态组合

## Constraints

- 手机：HarmonyOS 4.4.2
- 电视：华为智慧屏（支持安装第三方 APK，基于 Android TV 框架）
- 网络：中国移动宽带 + 运营商赠送的基础路由器（无 parental control）
- 无额外硬件（不买树莓派、不买旧手机）
- 远程控制（家长离家 40 公里）

## User Scenarios

1. **家长远程关电视**：点「关闭」→ 电视立即黑屏 → 孩子无法操作
2. **孩子重启电视**：电视开机 → TV App 自启 → 查询云端状态为"关闭" → 立即黑屏
3. **家长远程开电视**：点「打开」→ 电视恢复正常
4. **临时解锁**：家长点「临时解锁 5 分钟」→ 电视临时可用 → 5 分钟后自动锁定

## Premises

1. 华为智慧屏可以安装第三方 APK — 已确认
2. 电视 App 可以随开机自启动 — BOOT_COMPLETED 广播
3. 电视 App 通过全屏 Activity + LockTask 模式实现黑屏锁定（替代 SYSTEM_ALERT_WINDOW，后者在 TV 上受限）
4. 云服务器需要支持 WebSocket 长连接 → 使用轻量 VPS（非函数计算）
5. 手机和电视都能访问互联网（不需要在同一局域网）— 已验证

## Approaches Considered

### Approach A: 全自建（推荐）
手机 ArkUI App + Node.js VPS 云服务器 + 电视 Android APK
- 完整可控，可扩展
- 实时 WebSocket 通信，状态即时同步
- 开发周期约 2-3 周
- Pros：功能完整、不依赖第三方平台、可扩展多设备
- Cons：需要 VPS（最低 ~24元/月，或使用海外免费服务如 Render）

### Approach B: 极简版
手机 ArkUI App + 华为推送 Kit + 电视 Android APK
- 无需自建服务器
- 依赖华为推送服务
- 开发周期约 1-2 周
- Pros：无需运维服务器
- Cons：推送可能有延迟，功能受限

## Recommended Approach

Approach A：全自建方案。核心功能简单但需要实时可靠，VPS 月费极低。

## Component Specification

### 1. 手机 App（HarmonyOS ArkUI/ArkTS）

| 屏幕 | 功能 |
|------|------|
| 首页 | 电视状态（在线/离线/锁定）+ 大按钮「关闭电视」/「打开电视」 |
| 临时解锁 | 锁定状态下显示「临时解锁 5 分钟」按钮，服务器计时自动恢复 |
| 设备设置 | 首次使用输入电视上显示的配对码完成绑定 |

通信：WebSocket (wss) 连接到云服务器。断线自动重连。

### 2. 云服务器（Node.js + Socket.IO + SQLite）

部署：阿里云轻量应用服务器（~24元/月）或 Render/Fly.io 免费套餐。

注意：**不支持阿里云函数计算 / 腾讯云云函数**，因为需要 WebSocket 长连接。

| 功能 | 说明 |
|------|------|
| 设备注册 | 手机/电视注册，生成配对码 + AuthToken |
| 状态同步 | 手机设置状态 → 服务器存储 → 推送到电视 |
| 状态查询 | 电视开机后查询最新状态（支持 HTTP REST 作为备用） |
| 临时解锁 | 手机发起解锁指令，服务器端计时，到期自动恢复锁定 |
| 心跳检测 | 每 30 秒 ping/pong，60 秒无响应标记为离线 |

**设备配对流程：**
1. 电视 App 首次启动 → 向服务器注册 → 生成配对码 + SecretToken
2. 电视屏幕显示配对码（如 `TV-4821`）
3. 手机 App 输入配对码 → 服务器绑定手机和电视
4. 后续所有通信携带 AuthToken 鉴权

**数据库表：**

```sql
devices:    id, type(phone|tv), name, auth_token, paired_with, created_at
states:     device_id, state(open|closed), updated_at
unlocks:    device_id, expires_at, created_at
```

### 3. 电视 App（Android APK, Kotlin, minSdk 24）

| 模块 | 说明 |
|------|------|
| MainActivity | 首次启动显示配对码，配对后进入待机 |
| LockService | 前台 Service（通知栏常驻），维持 WebSocket 连接，心跳 30s |
| LockActivity | 全屏黑色 Activity，拦截所有按键（Back/Home/Volume），需密码退出 |
| BootReceiver | BOOT_COMPLETED 广播 → 启动 LockService |

**黑屏锁定实现（替代 SYSTEM_ALERT_WINDOW）：**
1. LockService 收到锁定指令 → 启动 LockActivity
2. LockActivity 全屏黑色，无退出按钮，拦截所有遥控器按键
3. 启用 LockTask 模式（设备策略控制器）防止 Home 键退出
4. 解锁时 LockService 关闭 LockActivity，电视恢复正常运行

**防卸载：**
- 使用设备管理员 (DeviceAdmin) 权限，防止孩子直接卸载
- 卸载需要先解除设备管理员，需家长远程操作或密码

**前台 Service 保活：**
- Android TV 对前台 Service 限制较松
- 使用 FOREGROUND_SERVICE_TYPE_CONNECTED 类型
- 通知栏显示"电视管控运行中"（TV 界面通知不显眼）

### 4. 通信协议

**认证：**
```json
// 所有消息携带 auth_token
{
  "type": "set_state",
  "device_id": "tv_xxx",
  "auth_token": "xxxx",
  "state": "closed"
}
```

**消息格式：**
```
手机 → 服务器: { type: "set_state", state: "closed" }
服务器 → 电视: { type: "state_update", state: "closed" }
电视 → 服务器: { type: "status_report", status: "locked" }
服务器 → 手机: { type: "tv_status", status: "locked" }
心跳: { type: "ping" } / { type: "pong" }
```

### 5. 部署架构

```
阿里云轻量应用服务器 (2核1GB, ~24元/月)
├── Node.js (PM2 保活)
├── Socket.IO (wss://)
├── SQLite (无需额外数据库)
└── Nginx (SSL 证书 + WebSocket 反向代理)
```

备选免费方案：Render.com 免费套餐（支持 WebSocket，每月 750 小时）

## Data Flow: 关机场景

```
1. 家长在手机 App 点「关闭电视」
2. 手机 → 云服务器 WebSocket: { type: "set_state", state: "closed", auth_token: "xxx" }
3. 服务器验证 auth_token，更新数据库 state = "closed"
4. 服务器 → 电视 WebSocket: { type: "state_update", state: "closed" }
5. 电视 LockService 收到 → startActivity(LockActivity)
6. LockActivity 全屏黑色，LockTask 阻止退出
7. 电视 → 服务器: { type: "status_report", status: "locked" }
8. 服务器 → 手机: { type: "tv_status", status: "locked" }
9. 手机显示：电视已锁定 ✓

孩子重启电视：
1. BOOT_COMPLETED → BootReceiver → LockService 启动
2. LockService 建立 WebSocket 连接 → 查询当前状态
3. 服务器返回: { state: "closed" }
4. LockService → 立即启动 LockActivity
5. 电视黑屏，Home/Back 全部拦截 ✓

孩子强制停止 App：
1. 试图停止 LockService → 前台 Service 阻止
2. 试图卸载 → 设备管理员权限阻止
3. 重启 → 自动恢复
```

## Success Criteria

1. 手机点「关闭」→ 电视在 5 秒内黑屏 ✓
2. 重启电视 → 电视自动锁定，不再播放 ✓
3. 手机点「打开」→ 电视在 5 秒内恢复正常 ✓
4. 整个过程不需要在同一局域网 ✓
5. App 被强制停止后，重启电视后自动恢复 ✓
